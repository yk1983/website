예외발생과 회피

예외 발생시키기

프로그래밍을 하다보면 처리 프로세스 중에 고의적으로 예외를 발생시켜야 할 상황이 있습니다. 이 때 키워드 throw를 사용하면 예외를 발생시킬 수 있습니다.

  1. 먼저, 연산자 new를 이용해서 발생시키려는 예외 클래스의 객체를 만듭니다.
  2. 키워드 throw를 이용해서 예외를 발생시킵니다.

Syntax

Exception e = new Exception("고의로 발생시킨 예외");
throw e;  // 예외를 발생시킴

// 선언과 동시에 예외 발생
throw new Exception("고의로 발생시킨 예외");

ExceptionEx9.java

package kinco.java.std;

public class ExceptionEx9 {
	public static void main(String[] args) {
		try {
			Exception e = new Exception("고의로 발생시킵니다.");
			throw e;
			// 위의 두 줄을 한 줄로 사용
			// throw new Exception("고의로 발생시킵니다.");
		} catch (Exception e) {
			System.out.println("에러 메세지 : " + e.getMessage());
			e.printStackTrace();
		}
		System.out.println("프로그램이 정상 종료되었습니다.");
	}
}

Console

에러 메세지 : 고의로 발생시킵니다.
java.lang.Exception: 고의로 발생시킵니다.
	at kinco.java.std.ExceptionEx9.main(ExceptionEx9.java:6)
프로그램이 정상 종료되었습니다.

예외처리 컴파일과 실행

Exception클래스들(Exception클래스와 그 자손 클래스들)이 발생할 가능성이 있는 문장들에 대해 예외처리를 해주지 않으면 컴파일조차 되지 않습니다.
RuntimeException클래스(RuntimeException클래스와 그 자손 클래스들)에 해당하는 예외는 프로그래머가 실수로 발생하는 것들이기 때문에 예외처리를 강제하지 않습니다. 하지만 컴파일이 정상적으로 되어도 예외가 발생하면 비정상적으로 종료됩니다.

메서드에 예외 선언하기

예외를 처리하는 방법에는 try-catch문을 사용하는 것 외에 예외를 메소드에 선언하는 방법이 있습니다. 메소드에 예외를 선언하려면 메소드의 선언부에 키워드 throws를 사용해서 메소드 내에서 발생할 수 있는 예외를 적어주기만 하면 됩니다. 예외가 여러 개인 경우에는 콤마(,)로 구분하여 적어주면 됩니다.

Syntax

void method() throws Exception1, Exception2, ... ExceptionN {
	// 메소드의 내용
}

예외 선언

예외를 선언하면 선언된 예외뿐만 아니라 그 자식타입의 예외까지도 발생할 수 있다는 점에 주의해야 합니다.

예외 메서드의 throws에 명시하는 것은 예외가 발생한 메서드에서 예외처리를 하지 않고 자신을 호출한 메서드에게 예외를 넘겨줄 수 있지만 이것으로 예외처리가 된 것은 아니고 단순히 예외를 전달하는 역활만 합니다. 따라서 어느 한 곳에서든 반드시 try-catch문으로 예외처리가 되어야 합니다.

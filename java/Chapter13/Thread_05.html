스레드 그룹(thread group)

스레드 그룹은 서로 관련된 스레드를 그룹으로 다루기 위한 것으로, 폴더를 생성해서 관련된 파일들을 함께 넣어서 관리하는 것처럼 스레드 그룹을 생성해서 스레드를 그룹으로 묶어서 관리할 수 있습니다.
또한 폴더 안에 폴더를 생성할 수 있듯이 스레드 그룹에 다른 스레드 그룹을 포함시킬 수 있습니다.
* 스레드 그룹은 보안상의 이유로 도입된 개념으로, 자신이 속한 스레드 그룹이나 하위 스레드 그룹은 변경할 수 있지만 다른 스레드 그룹의 스레드를 변경할 수 없습니다.

ThreadGroup의 생성자와 메서드

ThreadGroup(String name)
: 지정된 이름의 새로운 스레드 그룹을 생성합니다.
ThreadGroup(ThreadGroup parent, String name)
: 지정된 스레드 그룹에 포함되는 새로운 스레드 그룹을 생성합니다.
int activeCount()
: 스레드 그룹에 포함된 활성상태에 있는 스레드의 수를 반환합니다.
int activeGroupCount()
: 스레드 그룹에 포함된 활성상태에 있는 스레드 그룹의 수를 반환합니다.
void checkAccess()
: 현재 실행중인 스레드가 스레드 그룹을 변경할 권한이 있는지 체크합니다. 만일 권한이 없다면 SecurityException을 발생시킵니다.
void destory()
: 스레드 그룹과 하위 스레드 그룹까지 모두 삭제합니다. 단, 스레드 그룹이나 하위 스레드 그룹이 비어있어야 합니다.
int enumerate(Thread[] list)
int enumerate(Thread[] list, boolean recurse)
int enumerate(ThreadGroup[] list)
int enumerate(ThreadGroup[] list, boolean recurse)
: 스레드 그룹에 속한 스레드 또는 하위 스레드 그룹의 목록을 지정된 배열에 담고 그 개수를 반환합니다.
int getMaxPriority()
: 스레드 그룹의 최대우선순위를 반환합니다.
String getName()
: 스레드 그룹의 이름을 반환합니다.
ThreadGroup getParent()
: 스레드 그룹의 상위 스레드그룹을 반환합니다.
void interrupt()
: 스레드 그룹에 속한 모든 스레드를 interrupt
boolean isDaemon()
: 스레드 그룹이 데몬 스레드그룹인지 확인합니다.
boolean isDestroyed()
: 스레드 그룹이 삭제되었는지 확인합니다.
void list()
: 스레드 그룹에 속한 스레드와 하위 스레드그룹에 대한 정보를 출력합니다.
boolean parentOf(ThreadGroup g)
: 지정된 스레드 그룹의 상위 스레드그룹인지 확인합니다.
void setDaemon(boolean daemon)
: 스레드 그룹을 대몬 스레드그룹으로 설정 또는 해제 합니다.
void setMaxPriority(int pri)
: 스레드 그룹의 최대우선순위를 설정합니다.

스레드를 스레드 그룹에 포함시키려면 Thread의 생성자를 이용해야 합니다. 모든 스레드는 반드시 스레드 그룹에 포함되어 있어야 하기 때문에, 위와 같이 스레드 그룹을 지정하는 생성자를 사용하지 않은 스레드는 기본적으로 자신을 생성한 스레드와 같은 스레드 그룹에 속하게 됩니다.
예를 들어 main메서드를 수행하는 main이라는 이름의 스레드는 main스레드 그룹에 속하고, 가비지컬렉션을 수행하는 Finalizer스레드는 system스레드 그룹에 속합니다. 

ThreadGroup getThreadGroup() 스레드 자신이 속한 스레드 그룹을 반환합니다.
void uncaughtException(Thread t, throwable e)
스레드 그룹의 스레드가 처리되지 않은 예외에 의해 실행이 종료되었을 때, JVM에 의해 이 메서드가 자동적으로 호출됩니다.

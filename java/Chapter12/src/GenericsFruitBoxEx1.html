지네릭 클래스 Box<T>의 객체에는 한 가지 종류, T타입의 객체만 저장할 수 있습니다. 그리고 ArrayList를 이용해서 여러 객체를 저장할 수 있도록 하였습니다.

Box.java

package chapter12;

import java.util.ArrayList;

public class Box<T> {
	ArrayList<T> list = new ArrayList<T>();

	void add(T item) {
		list.add(item);
	}

	T get(int i) {
		return list.get(i);
	}

	int size() {
		return list.size();
	}

	public String toString() {
		return list.toString();
	}
}

Box<T> 객체를 생성할 때 참조변수와 생성자에 대입된 타입(매개변수화된 타입)이 일치해야 합니다. 두 타입이 상속관계에 있어도 마찬가지입니다.
단, 두 지네릭 클래스의 타입이 상속관계에 있고, 대입된 타입이 같은 것은 문제가 없습니다.
JDK1.7부터는 추정이 가능한 경우 타입을 생략할 수 있습니다.

FruitBoxEx1.java

package chapter12;

public class FruitBoxEx1 {
	public static void main(String[] args) {
		Box<Fruit> fruitBox = new Box<Fruit>();
		Box<Apple> appleBox = new Box<Apple>();
		// Box<Apple> appleBox = new Box<>(); // OK. JDK 1.7부터 생략가능
		Box<Toy> toyBox = new Box<Toy>();
		// Box<Grape> grapeBox = new Box<Apple>(); // 에러. 타입 불일치

		fruitBox.add(new Fruit());
		fruitBox.add(new Apple()); // OK. void add(Fruit item)

		appleBox.add(new Apple());
		appleBox.add(new Apple());
		// appleBox.add(new Toy()); // 에러. Box<Apple>에는 Apple만 담을 수 있음

		toyBox.add(new Toy());
		// toyBox.add(new Apple()); // 에러. Box<Toy>에는 Apple을 담을 수 없음

		System.out.println(fruitBox);
		System.out.println(appleBox);
		System.out.println(toyBox);
	}
}

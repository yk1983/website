표준입출력

표준입출력은 콘솔(console, 도스창)을 통한 데이터 입력과 콘솔로의 데이터 출력을 의미합니다.

표준입출력
System.in, System.out, System.err

자바에서는 표준 입출력(standard I/O)을 위해 3가지 입출력 스트림, System.in, System.out, System.err를 제공합니다. 이 들은 자바 어플리케이션의 실행과 동시에 사용할 수 있게 자동적으로 생성되기 때문에 개발자가 별도로 스트림을 생성하는 코드를 작성하지 않고도 사용이 가능합니다. 

System.in - 콘솔로부터 데이터를 입력받는데 사용합니다.
System.out - 콘솔로 데이터를 출력하는데 사용합니다.
System.err - 콘솔로 데이터를 출력하는데 사용합니다.

표준입력출력의 대상변경
setOut(), setErr(), setIn()

setOut(), setErr(), setIn()를 사용하면 입출력을 콘솔 이외에 다른 입출력 대상으로 변경하는 것이 가능합니다.
JDK 1.5부터는 Scanner클래스가 제공되면서 System.in으로부터 데이터를 입력받아 작업하는 것이 편리해졌습니다.

static void setOut(PrintStream out)
: System.out의 출력을 지정된 PrintStream으로 변경합니다.
static void setErr(PrintStream err)
: System.err의 출력을 지정된 PrintStream으로 변경합니다.
static void setIn(InputStream in)
: System.in의 입력을 지정된 InputStream으로 변경합니다.

RandomAccessFile

자바에서는 입력과 출력이 각각 분리되어 별도로 작업을 하도록 설계되어 있는데, RandomAccessFile만은 하나의 클래스로 파일에 대한 입력과 출력을 모두 할 수 있도록 되어 있습니다.
RandomAccessFile은 InputStream이나 OutputStream으로부터 상속받지 않고, DataInput인터페이스와 DataOutput인터페이스를 모두 구현했기 때문에 읽기와 쓰기가 모두 가능합니다.

DataInputStream은 DataInput인터페이스를 DataOutputStream은 DataOutput인터페이스를 구현했습니다. 이 두 클래스의 기본 자료형(primitive data type)을 읽고 쓰기 위한 메서드들은 모두 이 2개의 인터페이스에 정의되어 있는 것들입니다. 따라서 RandomAccessFile클래스도 DataInputStream과 DataOuputStream처럼 기본자료형 단위로 데이터를 읽고 쓸 수 있습니다.
또한 내부적으로 파일 포인터를 사용하는데, 입출력 시에 작업이 수행되는 곳이 바로 파일 포인터가 위치한 곳이 됩니다. 그래서 RandomAccessFile클래스는 파일의 어느 위치에나 읽기, 쓰기가 가능합니다. 현재 작업 중인 파일에서 파일 포인터의 위치를 알고 싶을 때는 getFilePointer()를 사용하면 되고, 파일 포인터의 위치를 옮기기 위해서는 seek(long pos)나 skipBytes(int n)를 사용하면 됩니다.

RandomAccessFile의 생성자와 메서드

RandomAccessFile(File file, String mode)
RandomAccessFile(String fileName, String mode)
: 주어진 file에 읽기 또는 읽기와 쓰기를 하기 위한 RandomAccessFile인스턴스를 생성합니다. mode의 값은 "r", "rw", "rws", "rwd"가 지정가능합니다.
"r"은 파일로부터 읽기(r)만을 수행할 때, "rw"은 파일에 읽기(r)와 쓰기(w), "rws"와 "rwd"는 기본적으로 "rw"와 같은데, 출력내용이 파일에 지연 없이 바로 쓰이게 합니다. "rwd"는 파일 내용만 "rws"는 파일의 메터정보도 포함합니다.
FileChannel getChannel()
: 파일의 파일 채널을 반환합니다.
FileDescriptor getFD()
: 파일의 파일 디스크립터를 반환합니다.
long getFilePointer()
: 파일 포인터의 위치를 알려줍니다.
long length()
: 파일의 크기를 얻을 수 있습니다.(단위 byte)
void seek(long pos)
: 파일 포인터의 위치를 변경합니다. 위치는 파일의 첫 부분부터 pos크기만큼 떨어진 곳입니다.(단위 byte)
void setLength(long newLength)
: 파일의 크기를 지정된 길이로 변경합니다.(단위 byte)
int skipBytes(int n)
: 지정된 수만큼 byte를 건너뜁니다.

* RandomAccessFile의 인스턴스를 "rw" mode로 생성할 때, 지정된 파일이 없으면 새로운 파일을 생성합니다.

용어사전

자바에서 사용되는 용어와 프로그래밍에서 사용되는 용어들을 정리하는 페이지입니다.

자바 (Java)

자바 (Java)는 썬 마이크로시스템즈의 제임스 고슬링(James Gosling)과 다른 연구원들이 개발한 객체 지향적 프로그래밍 언어입니다. 처음에는 가전제품 내에 탑재해 동작하는 프로그램을 위해 개발했지만 현재 웹 애플리케이션 개발에 가장 많이 사용하는 언어 가운데 하나이고, 모바일 기기용 소프트웨어 개발에도 널리 사용하고 있습니다. 자바의 개발자들은 유닉스 기반의 배경을 가지고 있었기 때문에 문법적인 특성은 파스칼이 아닌 C++의 조상인 C 언어와 비슷합니다.
자바를 다른 컴파일언어와 구분 짓는 가장 큰 특징은 컴파일된 코드가 플랫폼 독립적이라는 점입니다. 자바 컴파일러는 자바 언어로 작성된 프로그램을 바이트코드라는 특수한 바이너리 형태로 변환하고 바이트코드를 실행하기 위해서는 JVM(자바 가상 머신, Java Virtual Machine)이라는 특수한 가상 머신이 필요한데, 이 가상 머신은 자바 바이트코드를 어느 플랫폼에서나 동일한 형태로 실행시킵니다. 때문에 자바로 개발된 프로그램은 CPU나 운영 체제의 종류에 관계없이 JVM을 설치할 수 있는 시스템에서는 어디서나 실행할 수 있으며, 이 점이 웹 애플리케이션의 특성과 맞아떨어져 폭발적인 인기를 끌게 되었습니다.
2010년 썬 마이크로시스템즈가 오라클에 인수되어 오라클의 제품이 되었습니다.

객체지향 프로그래밍(OOP, Object Oriented Programming)

객체지향 프로그래밍이란 캡슐화, 다형성, 상속 을 이용하여 코드 재사용을 증가시키고, 유지보수를 감소시키는 장점을 얻기 위해서 객체들을 연결 시켜 프로그래밍 하는 것 입니다.
객체지향언어는 다음과 같은 특징을 갖습니다.

코드의 재사용성이 높습니다.
새로운 코드를 작성할 때 기존의 코드를 이용하여 쉽게 작성할 수 있습니다.
코드의 관리가 용이합니다.
코드간의 관계를 이용해서 적은 노력으로 쉽게 코드를 변경할 수 있습니다.
신뢰성이 높은 프로그래밍을 가능하게 합니다.
제어자와 메서드를 이용해서 데이터를 보호하고 올바른 값을 유지하도록 하며, 코드의 중복을 제거하여 코드의 불일치로 인한 오작동을 방지할 수 있습니다.

객체

표현의 모호성이 존재하지만 사전적인 정의는 실제로 존재하는 것, 우리 생각하고 표현할 수 있는 모든 대상입니다.
모델링의 관점에서는 명확한 의미를 정의한 대상 또는 개념이며, 프로그래머 관점에서는 객체는 클래스에서 정의된 메모리에 생성된 변수라고 할 수도 있습니다.
또한 소프트웨어 개발의 관점에서 본다면 어떤한 시점에 객체 상태를 나타내는 데이터와 해당 데이터를 처리하고 참조하는 동작을 의미하는 메서드(함수)를 모아놓은 '데이터+메서드' 형태의 소프트웨어 모듈로 정의할 수 있으며 객체지향 프로그래밍 관점에서의 객체는 데이터와 함수를 속성(attribute)과 메서드(method) 용어로 구현할 수 있습니다.

클래스 (class)

클래스는 객체를 생성하는데 사용하며, 객체를 정의해놓은 것 또는 객체를 만들어내기 위한 설계도, 틀이라고 정의할 수 있습니다.

변수 (variable)

수학에서 변수를 변하는 수라고 정의하지만 프로그래밍 언어에서의 변수(variable) 란 데이터(data)를 저장하기 위해 프로그램에 의해 이름을 할당받은 메모리 공간을 의미합니다. 즉, 변수란 데이터(data)를 저장할 수 있는 메모리 공간을 의미하며, 하나의 값을 저장할 수 있는 기억공간을 말합니다.
이렇게 저장된 값은 변경될 수 있습니다. 변수에는 하나의 변수에는 단 하나의 값만 저장할 수 있으므로, 새로운 값을 저장하면 기존의 값은 사라집니다.
변수의 타입에는 기본형과 참조형이 있습니다. 기본형 변수는 정수형( byte, short, int, long), 실수형( float, double), 논리형( boolean) 세 가지 분류로 나누어집니다. 기본형 변수를 제외한 변수를 참조형 변수라합니다.

상수 (constant)

상수는 변수와 마찬가지로 값을 저장할 수 있는 공간이지만 변수와 달리 한번 값을 저장하면 다른 값으로 변경할 수 없습니다. 자바에서 상수를 선언하는 방법은 변수와 동일하며 단지 타입 앞에 키워드 final을 붙여주기만 하면 됩니다. 그리고 상수는 반드시 선언과 동시에 초기화해야 합니다.

리터럴 (literal)

우리가 알고 있는 상수로써 그 자체로 값을 의미하는 것입니다. 프로그래밍에서 상수를 한번 값을 한 번 저장하면 변경할 수 없는 저장공간으로 정의하였기 때문에 이와 구분하기 위해 상수를 다른 이름으로 불러야 해서 리터럴이라는 용어를 사용합니다.

생성자 (constructor)

자바에서 생성자는 객체지향 프로그래밍에서 객체를 생성할 때, 인스턴스 변수를 원하는 값으로 초기화를 담당합니다. 즉, 생성자는 객체가 처음 생성될 때 호출되어 멤버 변수를 초기화하고, 필요에 따라 자원을 할당합니다.
생성자는 메소드와 비슷한 형태를 하고 있지만 생성자는 자바 클래스의 멤버가 아니므로 상속과 같은 멤버가 가지는 특성을 가지고 있지 않습니다.
생성자는 다음과 같은 특징을 가집니다.

  • 생성자의 이름은 클래스의 이름과 같아야 합니다.
  • 생성자는 리턴타입이 없으며 리턴 값도 없습니다.
  • 생성자는 인스턴스가 생성될 때 한번 호출됩니다.
  • 생성자는 클래스의 상속관계에서 초기화를 수행할 수 있도록 부모 클래스의 생성자를 호출할 수 있습니다.

메서드 (method)

메서드는 특정 작업을 수행하는 일련의 문장들을 하나로 묶은 것으로 기본적으로 수학에서의 함수와 비슷하며, 어떤 값을 입력하면 이 값으로 작업을 수행해서 결과를 반환합니다.
메서드의 장점은 다음과 같습니다.

높은 재사용성(reusability)
한 번 만들어 놓은 메서드는 몇 번이고 호출하여 사용할 수 있습니다.
중복된 코드의 제거
반복되는 문장들을 묶어서 하나의 메서드로 작성해 놓으면, 반복되는 문장들 대신 메서드를 호출하는 한 문장으로 대체할 수 있습니다.
프로그램의 구조화
문장들을 작업단위로 나눠서 여러 개의 메서드에 담아 프로그램의 구조를 단순화 시킬 수 있습니다.

JVM의 메모리 구조

메서드 영역
클래스 정보와 클래스 변수가 저장되는 곳입니다.
호출스택 (stack)
메서드의 작업공간으로 메서드가 호출되면 메서드 수행에 필요한 메모리 공간을 할당받고 메서드가 종료되면 사용하던 메모리를 반환합니다.
힙 (heap)
인스턴스가 생성되는 공간으로 new연산자에 의해서 생성되는 배열과 객체는 모두 여기에 생성됩니다.

클래스변수 (class variable)

클래스의 멤버변수로 자바에서는 문법적으로 전역변수를 지원하지 않지만 static 키워드를 이용해 전역변수처럼 이용할 수 있습니다. 클래스의 객체를 생성할 때 메모리에 생성(heap 메모리에 생성)되고, 변수의 초기화 지원으로 초기화 하지 않을 경우 기본값으로 초기화 됩니다. 가비지 컬렉터(Garbage Collector)에 의해 메모리에서 소멸됩니다.

지역변수 (local variable)

메서드 내에 선언된 변수들은 그 메서드 수행시 메모리에 생성(stack 메모리에 생성)되고, 메서드 내에서만 사용할 수 있으므로 서로 다른 메서드라면 같은 이름의 변수를 선언해도 됩니다. 메소드 종료시 메모리에서 소멸됩니다.

인스턴스변수 (instance variable)

클래스 영역에 선언되며, 클래스의 인스턴스를 생성할 때 만들어집니다. 인스턴스는 독립적인 저장공간을 가지므로 서로 다른 값을 가질 수 있으며, 인스턴스마다 고유한 상태를 유지해야 하는 속성의 경우 인스턴스변수로 선언합니다.

가비지 컬렉터 (Garbage Collector)

프로그램을 실행하다보면 가비지(Garbage)가 발생하게 됩니다. 가비지는 정리되지 않은 메모리, 유효하지 않은 메모리 주소를 말합니다. 즉, 가비지 컬렉터는 메모리가 부족할 때 쓰레기(가비지) 값를 정리해주는 프로그램을 말합니다.
JVM은 메모리를 부여받고 열심히 프로그램들을 실행하다가 메모리가 부족해지는 순간이 오면 OS에게 추가로 메모리를 더 요청하게 되며, 바로 이 메모리를 더 달라고 요청하는 때에 가비지 컬렉터(Garbage Collector)가 실행됩니다. 또는 서버 프로그램인 경우에는 24시간 내내 돌아가는데, 이 때에는 JVM이 한가할 때(idle time) 가비지 컬렉터가 실행되어 쓰레기 값들을 정리해주게 됩니다.

인스턴스 (instance)

인스턴스란 클래스에 속하는 각각의 객체로 하나의 클래스에서 생성된 객체를 말합니다. 클래스는 추상적인 개념으로 클래스에서 만들어낸 실제 객체를 생성하는 것을 인스턴스화(instantiate) 라고 합니다. 자바는 하나의 클래스로부터 여러 개의 인스턴스를 생성할 수 있습니다.

Map

Map 컬렉션

Map 인터페이스는 Collection 인터페이스와는 다른 저장 방식을 가집니다.
Map 인터페이스를 구현한 Map 컬렉션 클래스들은 키와 값을 하나의 쌍으로 저장하는 방식(key-value)을 사용하며, 요소의 저장 순서를 유지하지 않습니다. 여기서 키(key)란 실질적인 값(value)을 찾기 위한 이름의 역할을 합니다. 따라서 키(Key)는 중복값을 허용하지 않고, 값(value)는 중복값을 허용합니다.

대표적인 Map 컬렉션 클래스에 속하는 클래스는 다음과 같습니다.

  • HashMap<K, V>
  • Hashtable<K, V>
  • TreeMap<K, V>

HashMap

HashMap은 Map을 구현하므로 키(Key)와 값(Value)을 묶어서 하나의 데이터(entry)로 저장한다는 특징을 갖습니다. 그리고 JDK 1.2부터 제공된 HashMap 클래스는 해시 알고리즘(hash algorithm)을 사용하여 검색 속도가 매우 빠릅니다. HashMap에 저장된 데이터를 하나의 키로 검색했을 때 유일해야합니다.

HashMap의 생성자와 메서드

HashMap()
: HashMap 객체를 생성합니다.
HashMap(int initialCapacity)
: 지정된 값을 초기용량으로 하는 HashMap 객체를 생성합니다.
HashMap(int initialCapacity, float loadFactor)
: 지정된 초기용량과 load factor의 HashMap 객체를 생성합니다.
HashMap(Map m)
: 지정된 Map의 모든 요소를 포함하는 HashMap 객체를 생성합니다.
void clear()
: HashMap에 저장된 모든 객체를 제거합니다.
Object clone()
: 현재 HashMap을 복사해서 반환합니다.
boolean containsKey(Object key)
: HashMap에 지정된 키(key)가 포함되어있는지 알려줍니다. (포함되어 있으면 true를 반환합니다.)
boolean containsValue(Object value)
: HashMap에 지정된 값(value)가 포함되어있는지 알려줍니다. (포함되어 있으면 true를 반환합니다.)
Set entrySet()
: HashMap에 저장된 키와 값을 엔트리(키와 값의 결합)의형태로 Set에 저장해서 반환합니다.
Object get(Object key)
: 지정된 키(key)의 값(객체)을 반환합니다. 못찾으면 null을 반환합니다.
Object getOrDefault(Object key, Object defaultValue)
: 지정된 키(key)의 값(객체)을 반환합니다. 키를 못찾으면, 기본값(defaultValue)로 지정된 객체를 반환합니다.
boolean isEmpty()
: HashMap이 비어있는지 알려줍니다.
Set keySet()
: HashMap에 저장된 모든 키가 저장된 Set을 반환합니다.
Object put(Object key, Object value)
: 지정된 키와 값을 HashMap에 저장합니다.
void putAll(Map m)
: Map에 저장된 모든 요소를 HashMap에 저장합니다.
Object remove(Object key)
: HashMap에서 지정된 키로 저장된 값(객체)를 제거합니다.
Object replace(Object key, Object value)
: 지정된 키의 값을 지정된 객체(value)로 대체합니다.
boolean replace(Object key, Object oldValue, Object newValue)
: 지정된 키와 객체(oldValue)가 모두 일치하는 경우에만 새로운 객체(newValue)로 대체합니다.
int size()
: HashMap에 저장된 요소의 개수를 반환합니다.
Collection values()
: HashMap에 저장된 모든 값을 컬렉션의 형태로 반환합니다.

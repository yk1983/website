Set

Set 컬렉션

Set 인터페이스를 구현한 모든 Set 컬렉션 클래스는 요소의 저장 순서를 유지하지 않으며, 같은 요소의 중복 저장을 허용하지 않습니다.
대표적인 Set 컬렉션 클래스에 속하는 클래스는 다음과 같습니다.

  • HashSet<E>
  • TreeSet<E>

해시 알고리즘(hash algorithm)

해싱이란 해시함수(hash function)을 이용해서 데이터를 해시테이블(hash table)에 저장하고 검색하는 기법을 말합니다. 해시함수는 데이터가 저장되어 있는 곳을 알려 주기 때문에 다량의 데이터 중에서도 원하는 데이터를 빠르게 찾을 수 있습니다.
해싱에서 사용하는 자료구조는 배열과 연결 리스트의 조합으로 구현되어 있습니다. 저장할 데이터의 키를 해시함수에 넣으면 배열의 한 요소를 얻게 되고, 다시 그 곳에 연결되어 있는 연결 리스트에 저장됩니다.

HashSet

HashSet은 Set인터페이스를 구현한 가장 대표적인 컬렉션이며, Set인터페이스의 특징대로 HashSet은 중복된 요소를 저장하지 않습니다.
HashSet에 새로운 요소를 추가할 때는 add메서드나 addAll메서드를 사용하고, 이미 저장되어 있는 요소나 중복된 요소를 추가하고자 한다면 이 메서드들은 false를 반환하므로써 중복된 요소이기 때문에 추가에 실패했다고 알려줍니다.
만약 저장순서를 유지하고자 한다면 LinkedHashSet을 사용해야 합니다.

HashSetLotto.java

package kinco.java.std;
						
import java.util.*;

public class HashSetLotto {

	public static void main(String[] args) {
		// Set 객체를 생성
		Set<Integer> set = new HashSet<>();
		
		while (set.size() < 6) {
			int num = (int) ((Math.random() * 45) + 1);
			set.add(num);
		}
		// List 객체를 생성, set을 담습니다.
		List<Integer> list = new LinkedList<>(set);
		// 정렬
		Collections.sort(list);
		System.out.println(list);
	}
	
}

이진 검색 트리(BST, binary search tree)

이진 검색 트리(binary search tree)는 다음과 같은 특징을 가집니다.

  • 모든 노드는 최대 두 개의 자식노드를 가질 수 있습니다.
  • 각 노드에 값이 있습니다.
  • 각 노드의 키값은 모두 달라야 합니다. (중복된 노드가 없음)
  • 왼쪽 자식노드의 값은 부모노드의 값보다 작습니다.
  • 오른쪽 자식노드의 값은 보모노드의 값보다 큽니다.
  • 노드의 추가 삭제에 시간이 걸립니다. (비순차적인 저장)
  • 검색(범위검색)과 정렬에 유리합니다.

TreeSet

TreeSet은 이진 검색 트리(binary search tree)라는 자료구조의 형태로 데이터를 저장하는 컬렉션 클랙스입니다.
이진 검색 트리는 정렬, 검색, 범위검색(range search)에 높은 성능을 보이는 자료구조이며 TreeSet은 이진 검색 트리의 성능을 향상시킨 레드-블랙 트리(Red-Black tree)로 구현되어 있습니다.
그리고 Set인터페이스를 구현했으므로 중복된 데이터의 저장을 허용하지 않으며 정렬된 위치에 저장하므로 저장순서를 유지하지도 않습니다.

TreeSetLotto.java

package kinco.java.std;

import java.util.*;

public class TreeSetLotto {

	public static void main(String[] args) {
		// Set 객체를 생성
		Set<Integer> set =  new TreeSet<>();
		
		while (set.size() < 6) {
			int num = (int) ((Math.random() * 45) + 1);
			set.add(num);	
		}
		// 이미 정렬된 상태이므로 따로 정렬이 필요없음
		System.out.println(set);
	}
}

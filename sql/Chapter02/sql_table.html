TABLE

테이블은 데이터베이스의 기본적인 데이터 저장 단위 입니다. 데이터베이스 테이블은 사용자가 접근 가능한 모든 데이터를 보유하며 레코드와 컬럼으로 구성 됩니다. 관계형 데이터베이스가 아닌 예전의 데이터베이스 용어에서는 파일과 테이블, 필드와 컬럼 그리고 레코드와 행이 동일시 되었습니다.
테이블은 시스템내에서 독립적으로 사용되길 원하는 엔티티를 표현할 수 있습니다. 예를 들면, 회사에서의 직원이나 고객, 제품에 대한 주문은 테이블로 표현 가능합니다. 테이블은 두 엔티티간의 관계를 표현할 수 있습니다. 즉, 테이블은 직원과 그들의 작업 혹은 고객과 제품과 주문과의 관계를 표현하는데 사용될 수 있습니다. 테이블내에 있는 ForeIgn Key(외래 키)는 두 엔티티 사이의 관계를 표현하는데 사용 됩니다.

  • 컬럼(colnum) : 테이블의 각 컬럼은 엔티티의 한 속성을 표현 합니다.
  • 행(ROW, 레코드) : 테이블의 데이타는 행에 저장 됩니다.

CREATE TABLE 문

CREATE TABLE 문은 데이터베이스에 새 테이블을 만드는 데 사용됩니다.

CREATE TABLE Syntax

CREATE TABLE [schema.]table_name
(
	column datatype
	[, column datatype ...]
)
[TABLESPACE tablespace]
[PCTFREE integer]
[PCTUSED integer]
[INITRANS integer]
[MAXTRANS integer]
[STORAGE storage-clause]
[LOGGING | NOLOGGING]
[CACHE | NOCACHE];

테이블 생성시 규칙

  • 테이블 이름을 지정하고 각 컬럼들은 괄호 "()" 로 묶어 지정합니다.
  • 테이블 이름과 컬럼은 항상 알파벳 문자로 시작해야 하며 문자(A ~ Z) , 숫자(0 ~ 9), 그리고 특수문자($, #, _)를 사용할 수 있습니다. (공백 사용 불가능)
  • 테이블의 컬럼 이름은 30자를 초과할수 없고, 예약어를 사용할수 없습니다.
  • 컬럼뒤에 데이터 타입은 꼭 지정되어야 합니다.
  • 각 컬럼들은 콤마","로 구분되고, 항상 끝은 세미콜론";" 으로 끝납니다.
  • 한 테이블 안에서 컬럼명은 중복될 수 없으며, 다른 테이블에서의 컬럼 이름과는 중복될 수 있습니다.

테이블 생성 예제

CREATE TABLE EMP (
--  (컬럼)    (데이터타입)   (제약조건)
    EMPNO     NUMBER       CONSTRAINT emp_pk_empno PRIMARY KEY
,   ENAME     VARCHAR2(20)
,   JOB       VARCHAR2(40)
,   MGR       NUMBER
,   HIREDATE  DATE
,   SAL       NUMBER
,   COMM      NUMBER
,   DEPTNO    NUMBER
);

CREATE TABLE DEPT (
    DEPTNO    NUMBER        CONSTRAINT dept_pk_deptno  PRIMARY KEY
,   DNAME     VARCHAR2(40)
,   LOC       VARCHAR2(50)
);

사용자가 소유한 모든 테이블 조회

-- USER_TABLES 데이터사전을 조회 하면 유저가 소유한 테이블을 확인 할 수 있습니다.
SELECT table_name FROM USER_TABLES;

ALTER TABLE 문

테이블의 컬럼은 ADD, MODIFY, DROP를 통해서 관리 할 수 있습니다.

ALTER TABLE - ADD COLUMN

ALTER TABLE table_name
ADD column_name datatype;

ALTER TABLE - DROP COLUMN

테이블의 열을 삭제하려면 다음 구문을 사용하십시오. (일부 데이터베이스 시스템에서는 열 삭제가 허용되지 않음) 

ALTER TABLE table_name
DROP COLUMN column_name;

ALTER TABLE - ALTER / MODIFY COLUMN

테이블에있는 열의 데이터 형식을 변경하려면 다음 구문을 사용합니다.

SQL Server / MS Access:

ALTER TABLE table_name
ALTER COLUMN column_name datatype;

MySQL / Oracle (버전 10g 이전):

ALTER TABLE table_name
MODIFY COLUMN column_name datatype;

Oracle (버전 10g 이상):

ALTER TABLE table_name
MODIFY column_name datatype;

테이블의 테이블스페이스 변경

오라클 버전 8i부터는 ALTER TABLE ~ MOVE TABLESPACE 명령어로 쉽게 테이블의 테이블스페이스를 변경 할 수 있습니다.

Syntax

ALTER TALBE table_name MOVE TABLESPACE tablespace_name;

DROP TABLE 문

DROP TABLE 문은 데이터베이스의 기존 테이블을 삭제하는 데 사용됩니다. 이 때 주의할 점은 테이블을 삭제하면 테이블에 저장된 완전한 정보가 모두 삭제된다는 것입니다.

DROP TABLE Syntax

DROP TABLE table_name;

TRUNCATE TABLE 문

TRUNCATE TABLE 문은 테이블을 삭제하지는 않지만 테이블 내부의 모든 데이터를 삭제합니다. 또한 TRUNCATE TABLE은 DDL명령이므로 롤백 데이터가 생성되지 않습니다. 롤백(되돌리기) 데이터가 생성이 되지 않는다는 것은 DELETE 문으로 데이터를 지우면 롤백명렁어로 복구 할 수 있지만, TRUNCATE 문으로 데이터를 삭제하면 롤백을 할 수가 없습니다.
삭제되는 것은 데이터 뿐만 아니라 행당 인덱스도 같이 삭제되며, 외래키가 참조중인 테이블은 TRUNCATE할 수 없습니다.
TRUNCATE 문을 사용하면 삭제 트리거가 실행되지 않습니다.

TRUNCATE TABLE Syntax

TRUNCATE TABLE [schema.]table_name;
